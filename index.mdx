## Why solid-query-state?

Managing state in URL search params manually is tedious — you have to parse strings, handle missing values, keep state in sync on navigation, and write boilerplate for every parameter type. `solid-query-state` handles all of this with a clean, type-safe API that feels like native Solid.js.

## How it works

<Steps>
  <Step title="Wrap your app">
    Add `QueryStateAdapter` at the root of your component tree to enable URL sync.

    ```tsx
    import { QueryStateAdapter } from "solid-query-state";

    render(
      () => (
        <QueryStateAdapter>
          <App />
        </QueryStateAdapter>
      ),
      document.getElementById("root")!,
    );
    ```
  </Step>
  <Step title="Declare URL state">
    Use `createQueryState` like you would `createSignal`. Pass a parser to define the type.

    ```tsx
    import { createQueryState, parseAsInteger } from "solid-query-state";

    const [page, setPage] = createQueryState("page", parseAsInteger.withDefault(1));
    ```
  </Step>
  <Step title="Read and write">
    Call the getter in JSX and the setter in event handlers. The URL updates automatically.

    ```tsx
    <button onClick={() => setPage((p) => p + 1)}>
      Page {page()}
    </button>
    ```
  </Step>
</Steps>

<Note>
  **Requirements:** Solid.js ^1 and TypeScript ^5. Install with `npm install solid-query-state`.
</Note>
