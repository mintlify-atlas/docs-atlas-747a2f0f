---
title: "URL key mapping"
description: "Map state variable names to different URL parameter names using the urlKeys option."
---

By default, solid-query-state uses your state key names directly as URL parameter names. Sometimes that's not what you want:

- You might prefer shorter URL params (`q` instead of `search`) for cleaner links.
- An existing API or legacy URL scheme may use different names than your code does.
- You may want descriptive variable names in code without verbose URLs.

The `urlKeys` option in `createQueryStates` lets you decouple the two.

## How `urlKeys` works

Pass a `urlKeys` object alongside your key map. Each entry maps a state key to the URL parameter name that should appear in the browser address bar:

```tsx
import { createQueryStates, parseAsInteger, parseAsString } from "solid-query-state";

const [state, setState] = createQueryStates(
  {
    search: parseAsString.withDefault(""),
    page: parseAsInteger.withDefault(1),
  },
  {
    urlKeys: {
      search: "q",
      page: "p",
    },
  },
);
```

With this configuration:
- `state().search` reads from `?q=…` in the URL
- `state().page` reads from `?p=…` in the URL
- Setting `setState({ search: "laptop" })` writes `?q=laptop` to the URL

Your component code stays readable (`state().search`) while the URL stays compact (`?q=laptop&p=2`).

## A concrete example

Here's a search component that uses `q` as the URL param but `search` as the variable name:

```tsx
import { createQueryStates, parseAsString } from "solid-query-state";

function SearchBar() {
  const [state, setState] = createQueryStates(
    {
      search: parseAsString.withDefault(""),
    },
    {
      urlKeys: { search: "q" },
    },
  );

  return (
    <input
      type="search"
      placeholder="Search…"
      value={state().search}
      onInput={(e) => setState({ search: e.currentTarget.value })}
    />
  );
}
```

Typing `"hello"` into the input produces `?q=hello` in the URL. Reading the code, the intent is clear: you're working with `search`. Reading the URL, it's compact: `?q=hello`.

## Omitting entries

You only need to include keys you want to rename. Keys not listed in `urlKeys` fall back to using the state key name directly:

```tsx
const [state, setState] = createQueryStates(
  {
    search: parseAsString.withDefault(""),
    category: parseAsString.withDefault("all"),
    page: parseAsInteger.withDefault(1),
  },
  {
    urlKeys: {
      search: "q",
      // category and page use their own names as URL params
    },
  },
);
// URL: ?q=laptop&category=electronics&page=2
```

<Tip>
  Keep internal variable names semantic and descriptive. Use `urlKeys` to handle URL constraints—legacy param names, character limits, or API conventions—without polluting your component code with those concerns.
</Tip>
